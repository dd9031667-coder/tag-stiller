# TagStiller

TagStiller — настольное приложение для безопасного заполнения тегов локальных
аудиофайлов по публичной странице альбома Casa Musica. Приложение сначала
показывает сопоставление и ничего не записывает без явного подтверждения.

## Возможности

- загрузка публичной страницы через Playwright/Chromium;
- резервный импорт сохранённого HTML и загрузка ранее сохранённого JSON;
- полноценный локальный режим редактирования без HTML и доступа к сайту;
- drag-and-drop сохранённого HTML и папки с аудиофайлами;
- MP3, FLAC, M4A/MP4, OGG и OPUS;
- распознавание `01`, `Track 01`, `CD1-01`, `1-01` и похожих имён;
- сопоставление по диску и номеру, проверка названия и длительности;
- зелёные, жёлтые и красные статусы, ручной выбор файла двойным щелчком;
- редактирование новых Artist, Title, Language, Dance Style и Dance Tempo;
- Title без служебного хвоста вида `(Slow Waltz 29)`: танец и темп
  сохраняются только в `DANCE_STYLE` и `DANCE_TEMPO`;
- выбор записываемых полей и исключение отдельных строк;
- необязательное безопасное переименование файлов по редактируемому шаблону;
- именованные пресеты шаблонов переименования с сохранением между запусками;
- обновление Windows-версии одной кнопкой из опубликованных GitHub Releases;
- необязательное переименование папки альбома по редактируемому шаблону;
- автоматическое использование `cover.jpg`, `cover.jpeg` или `cover.png` из
  папки альбома как встроенной обложки;
- извлечение и редактирование `Album Artist`, `Album Year` и `Album Label`;
- экспорт данных альбома в JSON/CSV;
- атомарная запись через временную копию файла;
- обязательный JSON-backup исходных поддерживаемых тегов и восстановление;
- необязательные полные копии аудиофайлов;
- русский интерфейс, журнал `~/.tagstiller/tagstiller.log`.

`DANCE_TEMPO` является пользовательским полем и не записывается в BPM.
`DANCE_STYLE` также хранится отдельно; запись в стандартный Genre включается
отдельной галочкой. Пустые данные сайта по умолчанию не стирают существующие.
Album Label записывается как `TPUB` в MP3, `LABEL` в Vorbis Comments и
пользовательское поле `LABEL` в MP4.

## Запуск на macOS

Нужен Python 3.12. В Terminal из папки проекта:

```bash
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -e .
python -m playwright install chromium
python -m app.main
```

Также можно выполнить `chmod +x run.sh` и запустить `./run.sh` (скрипт использует
доступный `python3`; рекомендуется убедиться, что это Python 3.12).

CLI для проверки без интерфейса:

```bash
python -m app.cli parse-html saved-page.html -o album.json
python -m app.cli match album.json "/путь/к/альбому" --tolerance 4
```

## Типичный сценарий

1. Вставьте URL и нажмите «Загрузить данные». Если сайт показывает проверку,
   откройте страницу обычным браузером, сохраните её как HTML и нажмите
   «Открыть HTML». Программа не обходит CAPTCHA или ограничения доступа.
2. Выберите папку и нажмите «Сканировать папку».
3. Проверьте таблицу. Жёлтые строки требуют внимания, красные не записываются.
   Двойной щелчок по колонке «Локальный файл» позволяет выбрать файл вручную.
   Если рядом найден `cover.jpg`, `cover.jpeg` или `cover.png`, его имя будет
   показано в строке выбора папки. Галочка «Записывать cover» включена по
   умолчанию.
4. Исправьте редактируемые значения и снимите галочки с ненужных строк/полей.
5. При необходимости включите переименование. Шаблон по умолчанию:
   `{disc_prefix}{track:02d} - {artist} - {title}`. Доступны также `{disc}`,
   `{album}` и `{year}`. Ручное изменение шаблона автоматически включает
   переименование; оно выполняется даже при отсутствии изменений тегов.
   Расширение файла сохраняется автоматически.
   Отдельная галочка переименовывает саму папку альбома. Шаблон по умолчанию:
   `{album} - {year}` (например, `Rimini Open Vol. 01 - 2000`). Его можно
   свободно изменить; доступны `{album}`, `{year}`, `{label}` и
   `{album_artist}`. Например, прежний формат задаётся как
   `{label} - {album} ({year})`. Существующая папка с таким именем никогда
   не перезаписывается.
6. Нажмите «Записать теги», проверьте сводку и подтвердите.
7. Backup JSON появится рядом с аудиофайлами. После переименования пути внутри
   backup автоматически обновляются. Исходная встроенная обложка тоже входит
   в backup. Кнопка восстановления принимает этот файл.

HTML и папку можно не выбирать кнопками, а перетащить прямо на главное окно.
Если они перетащены одновременно, сначала обрабатывается HTML, затем папка
автоматически сканируется и сопоставляется с альбомом. Папка, перетащенная без
загруженного альбома, открывается в локальном режиме.

Шаблон переименования можно сохранить под собственным именем, затем выбирать
из списка пресетов или удалить. Последний введённый шаблон автоматически
восстанавливается при следующем запуске приложения.

Кнопка **«Очистить всё»** возвращает программу в начальное состояние: очищает
URL, альбом, папку, таблицу, журнал и настройки тегов. Текущее поле шаблона
переименования и все сохранённые пресеты при этом остаются.

## Локальное редактирование без Casa Musica

1. Выберите папку с аудиофайлами.
2. Нажмите **«Редактировать без HTML»**.
3. Программа прочитает существующие теги и построит такую же preview-таблицу.
4. Исправьте Disc, Track, Artist, Title, Album, Album Artist, Year, Label,
   Language, Dance Style и Dance Tempo.
5. При необходимости включите обложку, переименование файлов или папки.
6. Нажмите **«Записать теги»** и подтвердите изменения.

Если номер трека отсутствует, ему временно назначается номер по порядку, а
строка помечается жёлтым. Повторяющиеся номера также получают жёлтый статус.
Перед записью такие значения можно исправить непосредственно в колонках
«Диск» и «Трек».

Если существующий тег Album пуст, название альбома автоматически берётся из
имени папки. Имя вида `Album Label - Album Name (Album Year)` разбирается на
три соответствующих поля.

## Тесты

```bash
python -m pip install -e ".[dev]"
python -m pytest -m "not live"
```

HTML fixture используется в обычных тестах, поэтому они не зависят от сайта.
Live-тесты помечаются `live` и по умолчанию не запускаются.

## Windows ZIP через GitHub Actions

Каждый push в ветку `main` автоматически собирает Windows ZIP, вычисляет его
SHA-256 и публикует или обновляет GitHub Release с версией из `pyproject.toml`.
Внутри ZIP находится папка с `TagStiller.exe`; переносить нужно всю папку, а
не только EXE.

Начиная с версии 1.1.0, следующую опубликованную версию можно установить кнопкой
**«Проверить обновления»**. Приложение скачивает ZIP из публичного GitHub
Release, проверяет SHA-256, закрывается, заменяет свою папку и запускается снова.
Сохранённые шаблоны и остальные настройки находятся в профиле Windows и при
обновлении не удаляются. Если папка приложения защищена от записи, перенесите
её в пользовательскую папку, например `Документы`.

Сборка выполняется на `windows-latest` с Python 3.12. Chromium устанавливается
в пакет Playwright до PyInstaller и включается в onedir-сборку. Workflow
проверяет наличие EXE и браузера перед упаковкой. Размер ZIP поэтому будет
значительным.

## Ограничения версии 1.0

- структура сайта может измениться; предусмотрены несколько признаков таблицы,
  но при неизвестной разметке понадобится обновление парсера;
- Cloudflare может потребовать интерактивную проверку — используйте локальный
  HTML, приложение её не обходит;
- восстановление возвращает поддерживаемые TagStiller поля; неизвестные
  пользовательские поля не удаляются и не изменяются;
- атомарность достигается записью временной полной копии рядом с оригиналом и
  `os.replace`; на запись временно требуется свободное место размером с файл;
- код и тесты кроссплатформенны, но конкретный Windows artifact следует считать
  проверенным только после успешного запуска Windows workflow.
